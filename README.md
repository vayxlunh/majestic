# Majestic Helper

Личный помощник для игры на Majestic RP: калькулятор доходности, трекер денег,
чек-лист по фракциям, напоминания и таблица цен. Данные сохраняются прямо
в браузере (localStorage) — ничего не отправляется на сервер.

## Публикация на GitHub Pages — без терминала

Сайт настроен так, что GitHub сам соберёт и опубликует его после загрузки
файлов. Локально ничего устанавливать не нужно.

### Шаг 1. Создай репозиторий

На github.com → **New repository** → назови, например, `majestic-helper` →
Public → Create repository (ничего не отмечай при создании).

### Шаг 2. Проверь имя в vite.config.js

Если назвал репозиторий НЕ `majestic-helper` — открой файл `vite.config.js`
(можно прямо в браузере, через "Add file" после загрузки, кнопка карандаша)
и поменяй строку:

```js
base: "/majestic-helper/",
```

на своё название, например:

```js
base: "/my-repo-name/",
```

### Шаг 3. Загрузи файлы

На странице пустого репозитория нажми **uploading an existing file**
(или "Add file" → "Upload files"). Перетащи туда ВСЕ файлы и папки из этого
архива (включая скрытые `.github` и `.gitignore` — если браузер их не видит
при перетаскивании папки, перетащи саму папку `.github` отдельно).
Внизу нажми **Commit changes**.

### Шаг 4. Включи Pages через GitHub Actions

Repository → **Settings** → **Pages** → в разделе "Build and deployment"
в поле **Source** выбери **GitHub Actions** (не "Deploy from a branch").

### Шаг 5. Подожди сборку

Вкладка **Actions** в репозитории покажет процесс — займёт около минуты.
Когда появится зелёная галочка, сайт будет доступен по адресу:

```
https://<твой-ник>.github.io/<название-репозитория>/
```

Дальше при любом изменении файлов в репозитории сайт будет пересобираться
и обновляться автоматически.

## Важно про хранение данных

Данные (трекер, чек-лист, напоминания, цены) хранятся в `localStorage`
браузера — они привязаны к конкретному браузеру на конкретном устройстве.
С телефона или другого компьютера будет "чистый" экземпляр без старых
записей.
